# Metodología RUP (Enfoque Pragmático)

Este directorio contiene los artefactos de las fases de Análisis y Diseño del sistema de gestión de tareas compartidas.

## Estructura

- **01-analisis/**: Enfoque agnóstico a la tecnología. Se centra en entender el "qué" debe hacer el sistema, definiendo los requisitos funcionales y casos de uso sin comprometerse con una implementación técnica específica.
- **02-diseño/**: Enfoque en la arquitectura final. Se centra en el "cómo" se implementará el sistema, detallando la arquitectura técnica, el modelo de datos y la interacción entre componentes.

## Propósito

El objetivo de mantener esta separación es asegurar que la lógica de negocio y las necesidades del usuario (Análisis) sean claras antes de tomar decisiones técnicas profundas que conformarán la arquitectura del sistema (Diseño).
